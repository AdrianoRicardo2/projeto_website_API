    Inicio

    Para esta requisição, o resultado esperado é:

    
                {
                "success": true,
                "message": "Pong",
                "data": {
                    "teste": "conexao"
                    }
                 }

    Esta API é baseada no modelo REST tendo como formato de dados o padrão JSON, Os métodos devem ser acionados exclusivamente via POST sob a codificação UTF-8 conforme o exemplo:

    --location --request POST 'https://minhagarantidora.vouchsolucoes.com.br/ws/api/Ping' \ --header 'Authorization: Bearer 99fa077d-d805-4e68-a357-5a4b752d89d6' \ --header 'Content-Type: application/json' \ --data-raw

    '{ "teste": "conexao" }'

    Método que retorna todos os dados pertinentes a unidade. Atente-se que um CPF pode ter mais de uma unidade, por isso o campo [unidade] é obrigatório neste método. Agora o campo [unidade] aceita apenas inteiro.

    Exemplo 1

    curl --location --request POST 'https://minhagarantidora.vouchsolucoes.com.br/ws/api/BuscaDadosUnidade' \ --header 'Authorization: Bearer 99fa077d-d805-4e68-a357-5a4b752d89d6' \ --header 'Content-Type: application/json' \ --data-raw '{ "unidade": 150 }'

    Exemplo 2

    curl --location --request POST 'https://minhagarantidora.vouchsolucoes.com.br/ws/api/BuscaDadosUnidade' \ --header 'Authorization: Bearer 99fa077d-d805-4e68-a357-5a4b752d89d6' \ --header 'Content-Type: application/json' \ --data-raw '{ "unidade": 150, "cpfcnpj": "823.546.438-10" }'
